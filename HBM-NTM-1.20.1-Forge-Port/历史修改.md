### 校准 block 坐标 OBJ 的动态原点补偿

- 问题: 旧版 `press_head.obj` 原始 X/Z 范围为 `-0.375..0.375`，当前 port 资源已迁成 block 坐标 `0.125..0.875`；动态渲染需要明确围绕方块中心补偿原点，不能散落在单台机器 renderer 中
- 修改内容: 新增 `ObjPartTransform`，提供 `IDENTITY` 和 `BLOCK_CENTER` 原点补偿；`ObjModelPart` 支持内建原点/缩放变换；锻压机 `PRESS_HEAD` 在 `ObjModelLibrary` 中声明为 `BLOCK_CENTER.withScale(0.99F, 1.0F, 0.99F)`，`BasicMachineRenderer` 只保留面向和动画位移
- 接口复查: 当前 OBJ 动画库已覆盖旧版核心接口面：additional model 注册、单部件声明、组合模型、`renderAll`、`renderPart`、`renderOnly`、`renderAllExcept`、`getPartNames`、BlockEntity 动画回调、block 坐标原点补偿
- 验证: 使用 `127.0.0.1:7890` 代理执行 `compileJava processResources --no-daemon` 通过

### 修正锻压机压头轻微顿挫

- 问题: 现代 `sendBlockUpdated(..., 3)` 带有邻居/重渲染语义，作为动画同步包偏重；同时客户端收到速度/燃料变化但压头目标值未变化的更新时，也会重置 `turnProgress`
- 修改内容: 压机活动同步改为 `Block.UPDATE_CLIENTS`，只保留客户端数据更新；客户端 `load` 中仅当 `loadedPress != syncPress` 时重置 `turnProgress = 2`，避免非压头位置变化打断当前插值
- 对照结论: 1.7.10 旧版 `networkPackNT` 是轻量 ByteBuf 同步并带重复包过滤，现代移植不要把带邻居更新的 block update 当成高频动画包使用
- 验证: 使用 `127.0.0.1:7890` 代理执行 `compileJava processResources --no-daemon` 通过
